# ADR-002: hooks 安装使用 cp 而非 symlink

- **Date**: 2026-04-20
- **Status**: Accepted
- **Deciders**: arcaneorion
- **Tags**: hook, install, upgrade, reliability

## Context

`/spec-vc adr-init` 需要把 `$SKILL_ROOT/hooks/` 下的 `prepare-commit-msg` 与 `commit-msg` 投放到目标项目的 `.git/hooks/` 目录。技术路径有两种:

- **cp(复制文件)**:每次 init 产生一份独立的 hook 文件,存在于 `.git/hooks/` 之中。项目 hook 与 skill 源文件解耦。
- **symlink(符号链接)**:`.git/hooks/commit-msg` 指向 `$SKILL_ROOT/hooks/commit-msg`。项目 hook 始终反映 skill 最新代码。

核心张力:**同步性** 与 **独立性** 不可同时满足。

- symlink 得到同步:skill 修了 bug,所有使用该 skill 的项目自动获得修复。代价是**脆弱性**——skill 目录一旦被移动/删除/重命名,所有项目的 hook 全部失效,且失效方式是"commit 时报 No such file"这种 cryptic 错误,排查成本高。
- cp 得到独立性:项目 hook 与 skill 无耦合,skill 消失不影响项目。代价是**漂移**——skill 升级后,已初始化的项目不会自动同步。

另一个约束:本 skill 的目标使用者(当前是 arcaneorion 本人)会在多个项目(Alice、spec-vc 自身、以及未来的 Agent 项目)并发使用,且 skill 代码仓库本身也在演化。如果选 symlink,skill 仓库一次 `git mv` 就可能让所有下游项目同时失效。

## Decision

我们将使用 **cp 策略** 安装 hooks:

1. `/spec-vc adr-init` 在 `.git/hooks/` 下创建 hook 文件的**独立副本**,不建立符号链接。
2. 每个 hook 文件顶部写入 `# Version: x.y.z` 指纹,用于事后对比。
3. 新增 `/spec-vc adr-upgrade` 命令,读取 `.git/hooks/` 与 `$SKILL_ROOT/hooks/` 两侧的版本指纹,显示 diff 并询问是否覆盖。升级是**显式且可取消**的,永远不自动进行。
4. upgrade 之前自动备份旧 hook 为 `.git/hooks/<name>.backup-<timestamp>`。

## Consequences

**积极**:
- 项目 hook 对 skill 零依赖,skill 被删/移动/重命名都不会破坏已初始化项目的 commit 流程
- 版本指纹 + 显式 upgrade 让使用者明确知道"这个项目用的是 spec-vc 哪个版本的 hook",可与 git 历史交叉比对
- 升级过程可取消、有备份,单机场景下的漂移可控

**消极**:
- skill 修了 bug 不会自动传播,使用者必须对每个已初始化项目手动运行 `/spec-vc adr-upgrade`
- 各项目的 hook 版本可能分叉(v0.1 / v0.2 / v0.3 并存),维护者需要在心里记住"哪个项目用的哪个版本"
- 版本指纹只是注释,使用者如果手工编辑 hook 但不改指纹,upgrade 对比会误判

**中性**:
- 引入 `/spec-vc adr-upgrade` 作为新的心智负担,但它替代的不是"自动同步",而是替代"手动 cp"——相比手工操作仍是净收益
- hook 文件被 git 管理的范围限定在 skill 仓库内,各下游项目的 `.git/hooks/` 天然不进入 git 追踪,漂移是"运行态"问题而非"源码态"问题

## Alternatives Considered

- **symlink**:被排除,理由见 Context——脆弱性风险高于同步性收益。在单用户多项目场景下,skill 仓库演化频率 > 下游项目使用频率,symlink 会把 skill 的每次变更变成下游的潜在故障。
- **cp + 自动 upgrade**(init 时记录 skill 版本,每次 commit 前检查是否过期,过期则提示):被排除,理由是增加每次 commit 的 I/O 开销,且"过期提示"在日常 commit 流中会被忽略。显式的 `/spec-vc adr-upgrade` 命令比隐式提示更可靠。
- **git hooksPath 配置**(用 `git config core.hooksPath` 指向 skill 目录):属于 symlink 的变种,相同风险,被同样排除。此外 `hooksPath` 无法为不同项目指定不同 skill 版本,无法支持多版本共存。
