# ADR-011: 移除 commit submit 阶段，简化为 prepare + hook 校验循环

- **Date**: 2026-05-08
- **Status**: Accepted
- **Deciders**: arcaneorion
- **Tags**: commit-protocol, simplification

## Context

当前 commit 提交协议采用 prepare/submit 两阶段模型（ADR-008 引入 TTY-bound submit，ADR-009 引入 PostToolUse hook，ADR-010 移除机械层），但 submit 阶段（TTY 检测 + 交互确认 + token 写入）仍然是仪式性防线——它阻止 AI 点击提交按钮，但不提供实质审查内容：

- token 机制（uuid+timestamp+300s TTL）是纯机械校验，与变更质量无关
- TTY 检测防的是 AI 通过 Bash 工具调用 submit，但 AI 可直接 `git commit`
- 人类审批通过"Press Enter"完成，没有实质审查
- 整个 submit 命令是多余的中间层

核心洞察：spec-vc 的设计初衷是确保每次提交都有决策追溯和验证闭环。防护应该落在变更质量上，而不是谁按了按钮上。质量检查（subagent 审计、plan stage、Spec 完整性）都可以在 commit-msg hook 里机械检查。

## Decision

移除 `commit submit` 命令、token 机制（write/validate/consume/TTL）、TTY 检测、交互确认、prepare_ts 时间戳文件。将提交协议简化为单路径：

1. `commit prepare`（AI 域）：Spec 就绪检查 → 写 commit-msg
2. AI 执行 subagent 审计，PostToolUse hook 记录 session log
3. `git commit`（直接提交），commit-msg hook 自动校验
4. 校验失败 → AI 修改 → 重新 `git add` + `git commit`（自然循环）

新增 hook 校验项：
- plan stage ≥ `implement-ready`（有 active change 且引用 [ADR-NNN] 时）
- Spec 完整性（非骨架）（引用 [ADR-NNN] 且有关联 Spec 时）

保留 `commit prepare`（spec 就绪检查+写 commit-msg）、PostToolUse subagent session 记录、ADR 引用校验+豁免规则、SPEC_VC_BYPASS 逃生口。

## Consequences

- 积极后果：提交流程从三阶段缩减为两步（prepare → git commit），用户体验大幅简化；hook 校验链更加实质（检查变更质量而非谁按了按钮）；消除了 token TTL 过期带来的提交失败问题
- 消极后果：AI 可直接执行 `git commit`，但 commit-msg hook 仍然会阻塞不合规的提交；用户失去了一个"最后看一眼"的交互确认机会
- 中性后果：SPEC_VC_BYPASS 的作用范围略微扩大（跳过 stage+spec 检查），但 Bypass 仍然是应急手段

## Alternatives Considered

- **保留 submit 但简化 token**：仍保留 TTY 检测但移除 TTL。解决 token 过期问题但不解决仪式性问题。
- **将 submit 改为纯确认（无 token）**：只剩"Press Enter"一步。但这提供的是虚假安全感——用户不会逐条审查变更。
- **在 hook 中增加代码级语义一致性检查**：需要 AI 级理解，不适合机械检查，由 subagent 审计负责更合适。

## Implementation Plans

- **Plan**: doc/arch/plans/ADR-011-plan-001.md
- **Summary**: 移除 commit submit 阶段和 token 机制，简化提交流程为 prepare + hook 校验循环。commit-msg hook 新增 plan stage 和 Spec 完整性检查。
- **Pre-Validation**: 当前代码结构清晰，目标明确：从 commit.py 删除 token 机制，从 cli.py 删除 submit 命令，从 hooks.py 重构校验链。三个模块的改动边界清楚，没有横切关注点需要额外处理。新逻辑（plan stage 检查、spec 完整性检查）都有现成的函数可以直接复用。风险可控。
- **Post-Validation**: 所有 69 个测试通过。代码改动包括：(1) commit.py 删除 write_commit_token/validate_and_consume_token/TOKEN_TTL_SECONDS/TOKEN_FILENAME/PREPARE_TS_FILENAME，新增 check_subagent_session。(2) hooks.py 用 check_subagent_session 替换 validate_and_consume_token，新增 _check_plan_stage 和 _check_spec_readiness_for_adr。(3) cli.py 删除 cmd_commit_submit、submit subparser、prepare-ts 写入逻辑，清理 import。(4) 测试更新：4 个旧测试更新错误消息和断言，删除 test_commit_prepare_writes_prepare_ts，新增 3 个 plan stage 测试。(5) SKILL.md 和 CLAUDE.md 更新提交流程描述。
