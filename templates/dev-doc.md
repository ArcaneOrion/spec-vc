## 日志实现

### 9.1 日志级别规范
| 级别 | 使用场景 | 示例 |
|------|----------|------|
| ERROR | 系统异常、需要人工介入 | 数据库连接失败、第三方 API 不可用 |
| WARN | 可恢复异常、降级触发 | 重试成功、熔断开启 |
| INFO | 关键业务事件 | 用户登录、订单创建、支付完成 |
| DEBUG | 调试信息、参数详情 | 请求参数、SQL 语句、缓存命中 |

### 9.2 必须记录的事件
<!-- 列出必须记录日志的业务事件 -->
| 事件 | 日志级别 | 必须字段 | 说明 |
|------|----------|----------|------|
| | INFO | trace_id, user_id, action | |

### 9.3 日志格式
```json
{
  "timestamp": "ISO8601",
  "level": "INFO",
  "trace_id": "uuid",
  "span_id": "uuid",
  "service": "service-name",
  "method": "function_name",
  "message": "human readable message",
  "context": {}
}
```
